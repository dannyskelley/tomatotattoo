Diagnosis
- Error type: CLI not found (exit code 127).
- Cause: The build script calls the Sass CLI (`sass`) but that binary is not available in the build environment ([lines 57-59](#L57-L59)). This usually means the npm package that provides the `sass` binary (e.g., the "sass" package) is not installed or not being installed during the Netlify build (missing from package.json, only present locally, or devDependencies omitted during install).

Solution
1. Verify the project declares the Sass package in package.json (so Netlify can install it). Check your repo package.json:
   - https://github.com/dannyskelley/tomatotattoo/blob/main/package.json
   - If "sass" is missing from dependencies/devDependencies, add it.

2. Add and commit the sass package (recommended)
   - Run locally:
```bash
npm install --save-dev sass
git add package.json package-lock.json
git commit -m "chore: add sass to devDependencies"
git push
```
   - This ensures the `sass` binary is present at build time in node_modules/.bin and the `sass` script will succeed.

3. If "sass" is already listed in package.json but Netlify still can't find it, you may be skipping devDependencies during install. Ensure Netlify installs devDependencies:
   - In Netlify, do not set NPM_CONFIG_PRODUCTION=true or NPM_FLAGS=--production. If you need to force devDependencies to install, set in netlify.toml or Netlify UI environment:
```toml
[build.environment]
NPM_CONFIG_PRODUCTION = "false"
```
   - Or clear NPM_FLAGS so it doesn’t include --production.

4. Quick alternative (not recommended long-term): use npx so the CLI runs even if not installed as a top-level dependency:
```json
"scripts": {
  "build:sass": "npx sass src/assets/sass:./src/assets/css"
}
```
   - But installing "sass" in package.json is preferred for reproducible builds.

After making the change, push to the repo to trigger a new Netlify build. The `sass: not found` error ([line 59](#L59)) should be resolved once the Sass CLI is installed.
