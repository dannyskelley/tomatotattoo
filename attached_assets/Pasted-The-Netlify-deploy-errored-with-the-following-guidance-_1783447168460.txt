Diagnosis
- Error type: MODULE_NOT_FOUND at runtime when Netlify runs the configured build.command.
- Cause: The build command calls node ./node_modules/@11ty/eleventy/cmd.cjs but that file does not exist in the installed node_modules. This can happen if:
  1) @11ty/eleventy isn't actually installed (package missing from package.json or install failed), or
  2) The package is installed but the CLI entrypoint path used in the build.command is incorrect for the installed Eleventy version (so cmd.cjs doesn't exist), or
  3) the earlier npm install step logged an npm internal error ([line 19](#L19)) which may have prevented the package from being installed properly.

Solution — steps to verify and fix
1. Verify Eleventy is declared and committed
   - Open your repository package.json and confirm @11ty/eleventy is present in dependencies or devDependencies:
     https://github.com/dannyskelley/tomatotattoo/blob/main/package.json
   - Locally run: npm ls @11ty/eleventy
     - If it is missing, add it and commit:
       - npm install --save-dev @11ty/eleventy
       - commit package.json and package-lock.json

2. Prefer a robust invocation rather than relying on a hard-coded cmd.cjs path
   - Replace the direct module path in netlify.toml with npx (or an npm script). npx will invoke the package’s binary using the package’s published bin config and avoids hardcoding internal file paths that can change between versions.

   Option A — Update netlify.toml build.command to use npx:
   - Change the command in netlify.toml from the failing line to this:
```toml
[build]
  command = "node ./node_modules/sass/sass.js src/assets/sass:./src/assets/css && ELEVENTY_ENV=PROD npx @11ty/eleventy"
  publish = "public"
```

   Option B — Add an npm script and use npm run build (recommended)
   - In package.json add a build script (adjust eleventy flags / input/output as your project requires):
```json
{
  "scripts": {
    "build": "node ./node_modules/sass/sass.js src/assets/sass:./src/assets/css && ELEVENTY_ENV=PROD eleventy"
  }
}
```
   - Then set Netlify to run: npm run build (update netlify.toml build.command accordingly):
```toml
[build]
  command = "npm run build"
  publish = "public"
```
   - Commit package.json and netlify.toml changes.

3. If npm showed internal errors during install (you had "npm error Exit handler never called!" in the logs), clear Netlify’s build cache and retry a fresh build:
   - In Netlify UI: Trigger a "Clear cache and deploy site" (or set an environment variable to force a fresh install).
   - Locally, verify you can run npm install without errors; if npm itself fails consistently, consider removing node_modules & package-lock and reinstalling, or try a different npm version.

4. Verify the fix
   - Once deployed, check the build log to confirm the build runs the eleventy CLI via npx or the script and that node_modules/@11ty/eleventy is present during the build (no MODULE_NOT_FOUND for cmd.cjs).

Notes
- I recommended verifying @11ty/eleventy exists in package.json before changing invocation paths.
- Using npx or an npm script avoids depending on internal file paths such as cmd.cjs which may differ by version and prevents this exact MODULE_NOT_FOUND failure.
