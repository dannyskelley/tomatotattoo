Diagnosis
- Error type: Missing CLI binary / script failure (exit code 127).
- Cause: The build script tries to run ./node_modules/.bin/sass (from the "build:sass" npm script) but that executable is not present in node_modules, so the shell returns "not found" ([line 58](#L58)). This happens when the "sass" package (Dart Sass CLI) is not installed or the install step failed. The earlier npm internal error ([line 17](#L17)-[line 20](#L20)) suggests the install step may not have completed cleanly.

Solution
1. Verify your repo actually lists the Sass package and that package.json is committed
   - Open your package.json in the repo to check devDependencies/scripts: https://github.com/dannyskelley/tomatotattoo/blob/main/package.json
   - Ensure "sass" (the Dart Sass CLI) is present under devDependencies or dependencies. If it is missing or package.json is not committed, add/commit it before changing build behavior.

2. If "sass" is missing, add it and commit
   - Locally run:
```bash
npm install --save-dev sass
git add package.json package-lock.json
git commit -m "chore: add sass CLI"
git push
```
   - This installs the CLI into node_modules/.bin so ./node_modules/.bin/sass will exist during the Netlify build.

3. If "sass" is already in package.json but the build still fails, the npm install step may have failed (note the npm internal error at [line 17](#L17)-[line 20](#L20)). Try these remedial steps locally and push changes:
   - Clear and reinstall dependencies:
```bash
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
git add package-lock.json
git commit -m "chore: refresh lockfile"
git push
```
   - If npm continues to throw "Exit handler never called!", try upgrading/downgrading npm or pinning Node/NPM for Netlify and re-deploy. See Netlify docs for changing Node versions: https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript

4. Make the build script more robust (optional but recommended)
   - Instead of hardcoding the path to the binary, use npm's resolution or npx so the script works even if the bin path varies. Update your package.json script:
```json
"scripts": {
  "build:sass": "npx sass src/assets/sass:./src/assets/css",
  "build:eleventy": "eleventy",
  "build": "npm run build:sass && npm run build:eleventy"
}
```
   - Commit and push the change.

5. Re-run the Netlify build
   - After ensuring sass is present in package.json and lockfile is up-to-date (and optionally updating the script), push and trigger a new build. The missing-binary error at [line 58](#L58) should be resolved.
