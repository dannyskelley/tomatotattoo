Solution

1) Fix the missing run-s (npm-run-all)
- Verify whether npm-run-all is present in package.json and that package.json/lockfile are committed:
  - View your repo package.json: https://github.com/dannyskelley/tomatotattoo/blob/main/package.json
  - If npm-run-all (or another package providing run-s) is not listed in devDependencies or dependencies, add it.
- To add it locally and commit:
```bash
npm install --save-dev npm-run-all
git add package.json package-lock.json
git commit -m "chore: add npm-run-all for run-s used in build script"
git push
```
- Alternatively, if you prefer not to add npm-run-all, change your package.json build script to not use run-s (for example, call the individual build:* scripts in sequence) and commit that change.

2) Resolve the Node/sharp engine mismatch (recommended)
- sharp@0.35.3 requires Node >=20.9.0 ([lines 13–16](#L13-L16)). Running on Node 18 can cause warnings or installation/runtime problems.
- Recommended: target Node 20.x on Netlify. You can set the Node version by adding an .nvmrc at repo root (or setting engines.node in package.json). Example .nvmrc:
```text
20
```
Commit that file:
```bash
echo "20" > .nvmrc
git add .nvmrc
git commit -m "ci: use Node 20 on Netlify"
git push
```
- After committing .nvmrc (or engines.node), Netlify will use the specified Node version on the next build. For full instructions see Netlify docs for changing Node versions: https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript

3) Rebuild
- After applying step 1 (and step 2 if you changed Node version), trigger a new deploy on Netlify. The build should find run-s and install sharp compatible with Node 20.

Optional alternatives
- If you cannot change Node on Netlify, downgrade sharp to a version that supports Node 18 (not recommended unless necessary).
- If npm shows other internal errors after changing Node, run a clean install locally (rm -rf node_modules package-lock.json && npm install) to ensure lockfile and modules are consistent, commit the lockfile, and redeploy.
