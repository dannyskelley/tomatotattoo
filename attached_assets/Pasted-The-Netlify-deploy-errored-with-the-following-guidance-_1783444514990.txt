The Netlify deploy errored, with the following guidance provided:

Diagnosis
- Relevant log lines: build starts at [#L54-L56], the missing binary error at [#L59], and the failing build exit at [#L65-L69].
  - [#L54-L56] shows Netlify ran npm run build which runs the sass script.
  - [#L59] shows the runtime error: "sass: not found".
  - [#L65-L69] shows the npm build returning exit code 127 and failing the deploy.

- Error type: runtime CLI not found (exit code 127).
- Root cause: the build script calls the sass CLI ("sass src/assets/sass:./src/assets/css") but the sass package/CLI is not installed (or not present in package.json so it wasn't installed), so the binary is not available during the Netlify build.

Solution
1) Verify package.json contains sass
- Confirm your repository's package.json lists sass in dependencies or devDependencies:
  https://github.com/dannyskelley/tomatotattoo/blob/main/package.json
- If sass is missing, add it. If it's present but not committed, commit the change so Netlify can install it.

2) Add sass and commit (recommended)
- Locally run:
```bash
npm install --save-dev sass
git add package.json package-lock.json
git commit -m "chore: add sass devDependency for build"
git push
```
- This ensures npm installs the sass CLI into node_modules/.bin so the npm script "sass ..." will find it during the Netlify build.

3) Alternative quick fix (not recommended long-term)
- Change the build script in package.json to invoke npx, which will run a local or on-demand install:
```json
"scripts": {
  "build:sass": "npx sass src/assets/sass:./src/assets/css",
  ...
}
```
- Commit the change and push. But adding sass as a devDependency is preferable for reproducible builds.

4) Re-run the Netlify build
- After committing, trigger a new deploy (push to main or re-run the deploy in Netlify). The sass binary will be available and the build should proceed past the previous failure.

Notes
- You do not need to change Node versions for this error (Netlify is using Node v20.20.2 already). If you do need to change Node versions in the future, see Netlify docs: https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript
